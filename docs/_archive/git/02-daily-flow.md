# Git Günlük Akış (Daily Workflow)

Bu doküman, bir projede **her gün** en çok kullandığın Git akışını anlatır:
değişiklik yaptım → kaydettim → GitHub’a gönderdim.

---

## 1) Ne işe yarar?
Git, kod değişikliklerini **takip etmeyi** ve istenirse **geri almayı** sağlar.
GitHub ise bu değişiklikleri **internette saklamanı** ve ekipçe çalışmanı sağlar.

---

## 2) Neden önemli?
- “Hangi dosyayı ne zaman değiştirdim?” sorusunun cevabı olur.
- Hata yapınca “bir önceki çalışan hâle” geri dönmeyi kolaylaştırır.

---

## 3) Ne zaman kullanılır?
- Kodda/dosyada bir değişiklik yaptıktan sonra,
- “Bu kadarı tamam” dediğinde,
- Başkalarıyla paylaşmak istediğinde.
